RAID (Redundant Array of Independent Disks)

  • RAID 0 - Distribute the data across the disks, no redundancy
  • RAID 1 - Mirror the data on the other disk, tolerate one disk failure
  • RAID 3 - Stripe the data across the disks at the byte level with a dedicated disk that store the parity information for all disks
  • RAID 4 - Similar to RAID 3 but stripe the data at the block level
  • RAID 5 - Stripe the data and parity information at block level across the disks
  • RAID 6 - Similar to RAID 5 but has 2 sets of parity information distributed across the disks
  • RAID 10 - Combine RAID 0 and RAID 1

Block level : blocks of 512 bytes or larger

Stripe : smallest addressable piece of data

Stripe width : number of disks in the array

Parity

Parity is the information that used to detect the error and recover the data

  • The bit value is determined by the number of 1's in the data blocks at the same position
  • When the number of 1's is odd, the bit value should be 1. Otherwise 0.
Data Value
Block1 1001 1110
Block2 0111 0010
Block3 1100 1001
Parity 0010 0101

Benefits of RAID

  • Increase capacity
  • Higher availability
  • Increase performance It's not a substitution of backup
